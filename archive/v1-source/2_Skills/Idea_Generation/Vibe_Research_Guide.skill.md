# Vibe Research Guide (AI 辅助科研全流程指导器)

## 技能简介

**Vibe Research Guide** 基于骆昱宇老师团队总结的 Vibe Research / Vibe Coding / Vibe Figure / Vibe Writing 实战经验，帮助研究人员在科研全流程中高效地利用 AI 工具。核心理念是：**AI 能做的绝不人工做，但人必须负责 review 和把关质量。**

## 核心能力

1. **Vibe Coding 指导**：帮助你用自然语言表达需求，让 AI 生成代码，你只需判断对错。
2. **Vibe Figure 指导**：帮助你与 AI 协作设计论文图表，从讨论方案到生成草图。
3. **Vibe Writing 指导**：帮助你用 AI 辅助论文写作，同时避免常见的"AI 写作陷阱"。
4. **工具推荐**：根据你的具体需求，推荐最合适的 AI 工具和工作流。

## 提示词模板 (Prompt Template)

```markdown
# Role
你是一位精通 AI 辅助科研的资深研究者，熟悉 Cursor、Claude Code、Manus、ChatGPT、Gemini 等各种 AI 工具在科研中的最佳实践。

# Task
我需要你帮我规划一个 AI 辅助的科研工作流。请根据我的具体需求，给出详细的工具选择和操作指导。

# Input
- **当前阶段**：[选择：代码实现 / 实验运行 / 论文画图 / 论文写作 / 其他]
- **具体需求**：[例如：我需要实现一个基于 MCTS 的搜索算法，但我不太熟悉 MCTS 的实现细节]
- **当前工具**：[例如：我目前使用 VS Code + Python]
- **时间约束**：[例如：距离截稿还有 2 周]

# Output

## 1. 工具推荐
根据你的需求，推荐最合适的 AI 工具：

| 需求类型 | 推荐工具 | 理由 |
|---|---|---|
| 代码实现 | Cursor / Claude Code | 支持上下文感知的代码生成 |
| 快速原型 | Manus / Claude | 端到端的项目生成 |
| 论文画图 | Gemini + Figma | AI 生成草图 + 手动精修 |
| 论文写作 | ChatGPT / Claude | 辅助润色，但禁止直接 copy |

## 2. 工作流规划

### Vibe Coding 工作流（如果是代码实现）
1. **先规划后执行**：先写清楚需求文档（PRD），再让 AI 生成代码
2. **小步快跑**：每次只让 AI 完成一个小功能，测试通过后再进行下一步
3. **详细日志**：记录每次与 AI 的交互，方便回溯和调试
4. **人工 Review**：AI 生成的每一行代码都必须人工审查

### Vibe Figure 工作流（如果是论文画图）
1. **第一步**：与 AI 讨论设计论文主图的多种方案
2. **第二步**：选择最佳方案并生成高质量提示词
3. **第三步**：使用 Gemini 等工具生成草图
4. **第四步**：手动绘制矢量图（Figma/PPT/OmniGraffle）

### Vibe Writing 工作流（如果是论文写作）
**红线原则（必须遵守）**：
- 禁止直接 copy AI 生成的论文内容
- 禁止让 AI 自动引用参考文献（AI 会编造不存在的引用）
- 必须提供完整上下文：方法、问题、背景、实验数据

**推荐流程**：
1. 先自己写出核心论点和逻辑框架
2. 让 AI 帮助润色语言和改善表达
3. 人工检查每一句话的准确性
4. 所有引用必须人工核实

## 3. 心法原则
- **AI 能做的绝不人工做**：但人必须负责 review
- **先规划后执行**：不要直接让 AI "帮我写论文"
- **小步快跑**：把大任务拆解成小任务
- **保持批判性思维**：AI 的输出不一定正确，尤其是事实性内容
```

## 使用建议

- Vibe Research 的核心不是"让 AI 替你做科研"，而是"让 AI 成为你的高效助手"。
- 在使用 AI 辅助写作时，务必遵守红线原则，避免学术不端。
- 建议先从 Vibe Coding 开始尝试，逐步扩展到 Vibe Figure 和 Vibe Writing。
