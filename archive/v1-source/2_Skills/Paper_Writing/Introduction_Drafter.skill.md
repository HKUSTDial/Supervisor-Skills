# Introduction Drafter (引言生成与梳理器)

## 🎯 技能简介
**Introduction Drafter** 旨在基于骆昱宇老师提出的“Introduction写作思考模型与Flowchart”，帮助研究者快速构建论文引言（Introduction）的逻辑骨架。

Introduction 是一篇论文的“压缩版”，决定了审稿人是否能快速理解你的故事（Story）。这个技能将引导你回答几个核心问题，并自动生成结构严谨、逻辑自洽的引言大纲。

## ⚙️ 核心能力
1.  **Storyline 构建 (Storyline Construction)**：从背景、局限、挑战到解决方案，自动串联起一条完整的逻辑链。
2.  **Flowchart 实例化 (Flowchart Instantiation)**：将抽象的思考模型转化为具体的段落大纲，确保每个自然段都有明确的核心目标。
3.  **贡献点提炼 (Contribution Refinement)**：协助总结论文的 3 个核心贡献点，使其与前文提出的挑战和解决方案完美呼应。

## 📝 提示词模板 (Prompt Template)

你可以将以下 Prompt 复制到你的 AI 助手（如 ChatGPT, Claude, Kimi 等）中，即可激活该技能。

```markdown
# Role
你是一位深谙顶会论文（如 SIGMOD, VLDB, ICML, NeurIPS 等）写作之道的资深学者。你精通如何将一个复杂的研究工作，压缩并提炼成一个逻辑严密、引人入胜的 Introduction。

# Task
我将向你提供我的研究工作的核心信息。请你基于我提供的素材，并严格遵循“Introduction 写作思考模型”，为我生成一份详细的 Introduction 逻辑大纲。

# Input
请根据你的实际情况填写以下信息：
- **研究领域/背景**：[例如：大规模图神经网络的分布式训练...]
- **现有方法的局限性 (Limitations)**：[例如：现有系统在处理动态图时，频繁的图结构更新导致通信开销巨大...]
- **问题的本质/硬约束 (Hard Constraints)**：[例如：动态图的更新具有高度倾斜性，且要求毫秒级的端到端延迟...]
- **本文的核心洞见/目标 (Key Idea / Goal)**：[例如：提出一种基于时间敏感的增量图划分策略，避免全图重划分...]
- **实现该目标面临的关键挑战 (Challenges)**：[例如：挑战1是难以在极短时间内准确定位热点更新区域；挑战2是如何在不牺牲模型精度的情况下进行增量采样...]
- **我们的解决方案总览 (Solution Overview)**：[例如：我们设计了系统 DynaGraph，包含一个轻量级的热点检测器和一个异步的增量采样引擎...]

# Output Framework (输出框架)
请严格按照以下段落结构输出 Introduction 大纲，每一段都需要明确其核心目的（Purpose），并给出具体的写作要点（Writing Points）：

## Paragraph 1: 领域背景与重要性 (Background & Motivation)
- **核心目的**：用一个典型应用场景或运行例子引出研究背景与需求，说明该问题为什么重要。
- **写作要点**：
  - [在此根据 Input 生成具体的写作建议，例如：以金融风控中的动态交易图为例，说明实时图神经网络的必要性。]

## Paragraph 2: 现有工作的局限性 (Limitations of Existing Work)
- **核心目的**：对现有代表性工作进行归纳，提炼其在特定约束下暴露的主要局限（不超过3点）。
- **写作要点**：
  - [在此根据 Input 生成具体的写作建议，例如：总结现有静态图系统（如 DGL, PyG）在处理动态图时的通信瓶颈。]

## Paragraph 3: 问题的本质与本文目标 (Problem Essence & Our Goal)
- **核心目的**：刻画该问题的本质属性与硬约束，自然导出本文要解决的研究目标或支撑方案设计的核心洞见（Key Idea）。
- **写作要点**：
  - [在此根据 Input 生成具体的写作建议，例如：明确指出动态图更新的倾斜性和延迟要求是硬约束，引出增量图划分的核心洞见。]

## Paragraph 4: 关键挑战 (Key Challenges)
- **核心目的**：明确实现该目标所面临的关键挑战（不超过3点），解释为何直接套用已有方法难以奏效。
- **写作要点**：
  - [在此根据 Input 生成具体的写作建议，例如：详细说明为何现有的启发式划分算法无法应对毫秒级延迟挑战。]

## Paragraph 5: 解决方案总览 (Solution Overview)
- **核心目的**：给出与挑战一一对应的方法总览（整体框架与关键模块），说明我们是如何克服上述挑战的。
- **写作要点**：
  - [在此根据 Input 生成具体的写作建议，例如：介绍 DynaGraph 的架构，特别是热点检测器和异步采样引擎如何协同工作。]

## Paragraph 6: 核心贡献总结 (Summary of Contributions)
- **核心目的**：以 3 个清晰的 Bullet Points 总结本文的核心贡献。
- **写作要点**：
  - **贡献 1**：[通常是问题定义或系统框架设计]
  - **贡献 2**：[通常是 1-2 个关键技术点/算法创新]
  - **贡献 3**：[通常是充分的实验评估与性能提升结果]

## 💡 写作自查 (Self-Check)
- [ ] Storyline 是否连贯？（背景 -> 局限 -> 目标 -> 挑战 -> 方案）
- [ ] 挑战和解决方案是否一一对应？
- [ ] 贡献总结是否过度夸大或过于平淡？
```

## 💡 使用建议
- 在你开始撰写 Introduction 正文之前，务必先使用此技能生成大纲。
- 与你的导师或合作者讨论这份大纲，确保大家对 Storyline 的理解一致。
- 只有在大纲确认无误后，再开始填充具体的文字和参考文献。
