# Tech Paper Template Filler (技术类论文逻辑链构建器)

## 技能简介

**Tech Paper Template Filler** 基于骆昱宇老师提出的"技术类 Full Paper 思考模板"，帮助研究人员在动笔写论文之前，系统性地梳理论文的完整逻辑链——从研究背景到贡献点，确保每一个环节都经得起审稿人的推敲。

通过使用这个技能，你可以在 30 分钟内完成一篇技术类论文的逻辑骨架搭建，避免写到一半才发现逻辑链断裂。

## 核心能力

1. **论文类型定位**：帮助你快速判断论文属于"Technique Paper（新方法解决既有问题）"还是"New Problem/Setting（新问题/新设定）"，从而确定写作主轴。
2. **逻辑链完整性检查**：按照 Flowchart 的逻辑链（研究背景 → Limitations → Key Idea/Our Goal → Challenges → 方法总览 → 贡献点），逐一检查每个环节是否完整、是否自洽。
3. **思考模板实例化**：将你的研究内容填入标准化的思考模板表格中，生成一份可直接用于指导论文写作的结构化文档。

## 提示词模板 (Prompt Template)

将以下 Prompt 复制到你的 AI 助手中，即可激活该技能。

```markdown
# Role
你是一位在 SIGMOD/VLDB/ICML/NeurIPS/ICLR 等顶会上发表过 40+ 篇论文的资深研究者和审稿人。你擅长帮助学生梳理论文的逻辑链，确保每一个环节都经得起审稿人的推敲。

# Task
我将向你提供一篇技术类论文的核心信息，请你帮我按照以下"Introduction 写作 Flowchart"的逻辑链，系统性地梳理论文的完整逻辑骨架，并填写思考模板表格。

# Introduction 写作 Flowchart 逻辑链
研究背景 → 研究前沿（现有方法）→ Limitations（不超过3点）→ Key Idea / Our Goal → Challenges（不超过3点）→ 方法总览 → 贡献点

# Input
- **研究领域**：[例如：Text-to-SQL / 图神经网络 / LLM Agent]
- **要解决的核心问题**：[例如：现有的 zero-shot Text-to-SQL 方法在复杂查询上准确率不足]
- **我们的方法/系统名称**：[例如：Alpha-SQL]
- **方法的核心思路（一句话）**：[例如：利用 MCTS 在 SQL 生成空间中进行搜索优化]
- **主要实验结果（如有）**：[例如：在 BIRD 和 Spider 上分别提升了 X% 和 Y%]

# Output Format
请严格按照以下思考模板表格输出，每个单元格都必须填写完整的内容：

## Step 1: 论文类型定位

首先判断这篇论文属于哪种类型，并确定写作主轴：

| 论文类型 | 写作主轴 | 是否匹配 |
|---|---|---|
| Technique Paper（新方法解决既有问题） | Key Idea / Mechanism | 是/否 |
| New Problem/Setting（新问题/新设定） | Our Goal / Problem Formulation | 是/否 |

## Step 2: 思考模板表格

| 逻辑阶段 | 你的论文内容 |
|---|---|
| **研究背景** | |
| 研究场景是什么？为什么重要？需要一个清晰的场景定义 + 研究动机 | [填写] |
| **研究问题分析与属性解读** | |
| 对现状或者现有最新方法的分析，总结局限性。一般总结不超过3个。 | |
| Limitation 1 | [填写：现有方法的第一个关键局限] |
| Limitation 2 | [填写：现有方法的第二个关键局限] |
| Limitation 3（如有） | [填写：现有方法的第三个关键局限，如果只有两个也可以] |
| **论文的 Novelty 和创新思路讨论** | |
| Key Idea / Our Goal | [填写：核心洞见或研究目标] |
| **挑战分析** | |
| Challenge 1 | [填写：实现目标面临的第一个关键挑战] |
| Challenge 2 | [填写：实现目标面临的第二个关键挑战] |
| Challenge 3（如有） | [填写：第三个关键挑战] |
| **方法总览** | [填写：整体框架与关键模块，与 Challenges 一一对应] |
| **贡献点** | [填写：3-4 个贡献点，包括问题定义、框架设计、关键技术、实验评估] |

## Step 3: 逻辑链自洽性检查

请检查以下几个关键连接是否自然、自洽：
1. Limitations → Key Idea/Our Goal：我们的创新思路是否直接回应了前面提出的局限性？
2. Key Idea → Challenges：挑战是否是实现 Key Idea 过程中自然产生的？
3. Challenges → 方法总览：方法的每个关键模块是否都对应了一个具体的挑战？
4. 方法总览 → 贡献点：贡献点是否完整覆盖了方法的所有关键创新？

如果发现逻辑链断裂，请明确指出并给出修复建议。
```

## 使用建议

- 在动笔写论文之前，先用这个技能梳理一遍逻辑链，可以节省大量返工时间。
- 如果 AI 在"逻辑链自洽性检查"中发现了断裂，说明你的论文 Story 还需要打磨。
- 你可以把 AI 生成的思考模板表格打印出来，贴在显示器旁边，作为写作时的参考。
- 建议配合 [Introduction Drafter](./Introduction_Drafter.skill.md) 技能一起使用：先用本技能梳理逻辑链，再用 Introduction Drafter 生成 Intro 草稿。
